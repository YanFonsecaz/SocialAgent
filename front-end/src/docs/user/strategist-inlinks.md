# Strategist Inlinks — Guia rápido

## Para que serve
O **Strategist Inlinks** analisa uma **URL principal** (seu conteúdo) e um conjunto de **URLs para análise** (outros conteúdos do seu site ou de páginas relacionadas) para identificar **oportunidades de links internos**.

Ele ajuda você a:
- encontrar pontos naturais do texto onde um link interno faria sentido;
- sugerir **âncoras** (o texto clicável) e o **URL de destino**;
- gerar um relatório com o que foi selecionado e o que foi rejeitado (com justificativas);
- visualizar versões do conteúdo (original / com links / modificado).

> Link interno bem colocado melhora navegação, distribuição de autoridade e pode ajudar SEO (quando feito com relevância e sem exagero).

---

## O que ele faz (passo a passo)
1. Você informa uma **URL principal** (o artigo/página onde o link será inserido).
2. Você informa **URLs para análise** (páginas candidatas a serem linkadas).
3. A ferramenta extrai texto e identifica trechos relevantes.
4. Ela escolhe oportunidades de links internos com base em relevância e regras de densidade.
5. Retorna:
   - itens **selecionados** (com âncora + destino);
   - itens **rejeitados** (com motivo);
   - versões do conteúdo para visualizar a aplicação dos links.

---

## Campos (Configuração)

### URL Principal
A página **onde** o link interno será inserido.

**Dicas:**
- Use uma URL pública e acessível (sem login).
- Quanto mais completo o texto, melhores as oportunidades.
- Prefira páginas com conteúdo editorial (posts, artigos, guias).

---

### URLs para Análise (uma por linha)
Lista de páginas candidatas a serem **destinos** do link interno.

**Como escolher bem:**
- use URLs do mesmo domínio (seu site) quando o objetivo for linking interno;
- priorize conteúdos que complementam o tema da URL principal;
- evite URLs muito parecidas entre si (concorrendo pela mesma âncora).

**Dicas práticas:**
- 5 a 20 URLs costuma ser um bom começo;
- coloque uma URL por linha, sem vírgulas.

---

## Resultados (o que você vai ver)

### Selecionadas
São as oportunidades aprovadas pela ferramenta. Cada item normalmente contém:
- **url**: o destino do link (para onde o usuário será levado)
- **sentence**: trecho/linha do texto onde a âncora aparece (contexto)
- **anchor**: o texto clicável sugerido

**Como usar:**
- valide se a âncora está natural (não forçada);
- valide se o destino realmente aprofunda o assunto;
- ajuste a âncora para ficar mais clara (ex: “clique aqui” → algo descritivo).

---

### Rejeitadas
Itens analisados que **não foram aprovados**, com o motivo.

Motivos comuns:
- baixa relevância com o contexto
- repetição (âncora/destino duplicados)
- densidade de links atingida (para não “poluir” o texto)
- bloqueios por regra (ex: já há link naquele bloco)

---

### Visualização (Original / Linked / Modified)
Você pode ver versões do conteúdo:
- **Original**: como está hoje
- **Linked**: com links inseridos (âncoras viram links)
- **Modified**: quando há ajustes de texto (quando necessário) para encaixar melhor o link

> Em geral, a melhor prática é inserir links com mínima alteração no texto, mantendo naturalidade.

---

## Boas práticas (recomendado)
- Use âncoras **descritivas** (ex: “seguro auto para elétricos”, não “saiba mais”).
- Linke para páginas que realmente completam o assunto (intenção de busca semelhante).
- Não exagere: muitos links por parágrafo pode piorar a leitura.
- Prefira inserir links em trechos onde a pessoa naturalmente “teria a próxima pergunta”.

---

## Limitações e cuidados
- Se a URL principal for muito curta, pode haver poucas oportunidades.
- Páginas com conteúdo dinâmico, paywall ou bloqueios podem ter extração incompleta.
- A ferramenta ajuda a sugerir, mas você ainda deve revisar antes de publicar (especialmente para tom/clareza e para evitar âncoras repetidas).

---

## Checklist rápido antes de aplicar
- [ ] O destino do link faz sentido para quem está lendo esse parágrafo?
- [ ] A âncora é natural e descreve o destino?
- [ ] Não há excesso de links muito próximos?
- [ ] O link não compete com outros links mais importantes no mesmo trecho?
- [ ] As URLs de destino estão corretas e acessíveis?
